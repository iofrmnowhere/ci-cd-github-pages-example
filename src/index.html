Ugliest UI — Case Study

The screenshot (included)

Why this UI is bad

  • Poor contrast and color choices that reduce readability.
  • Centered, non-hierarchical layout with no visual rhythm.
  • Confusing affordances (buttons and links that don't behave predictably).
  • Large hit targets that contrast awkwardly with surrounding text.
  • Poor accessibility: missing ARIA roles, keyboard traps, and low contrast text.

Proposed improvements

  1. Establish a typographic scale and hierarchy (clear headings, paragraph sizes).
  2. Improve color contrast and use a limited palette; test with contrast tools.
  3. Use clear affordances: primary/secondary buttons, consistent link styles.
  4. Make layout responsive and use spacing system for rhythm and balance.
  5. Improve accessibility — semantic HTML, ARIA where needed, and keyboard navigation.
